# rtw-examples

Test examples for [ready-to-work (rtw)](https://github.com/joey-lou/ready-to-work) — an architect loop for AI-driven development.

Each subdirectory is a self-contained example with a `TASK.md` specification. The `run.sh` script orchestrates rtw runs with a clean workspace per execution.

## Usage

```bash
# Run an example (creates a fresh workspace under <example>/runs/<timestamp>/)
./run.sh simple_api

# Pass extra rtw flags
./run.sh simple_api --max-iter 5 --backend cursor
```

## Structure

```
simple_api/
├── TASK.md              # Task specification (permanent)
└── runs/                # One directory per rtw run
    └── 20260318_120000/ # Each run IS the workspace
        ├── main.py      # Code generated by rtw
        ├── ...
        └── .rtw/        # RTW run metadata (plan, traces, summary)
```

## How it works

1. `run.sh <example>` creates a timestamped directory under `<example>/runs/`
2. The workspace starts clean — no leftover files from previous runs
3. `rtw run TASK.md -w <run_dir>` executes the architect loop in the fresh workspace
4. Both the generated code and `.rtw/` metadata are preserved for review and git tracking
